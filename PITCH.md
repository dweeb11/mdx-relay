# MDX Relay

## What is this?

A desktop Obsidian plugin that converts an approved note and its inline raster images into repository-specific MDX, then performs a narrow reviewed Git commit and verified push.

## Why does this exist?

The current `dpw-mind-net` importer works, but it is tied to one repository and one command-line workflow. MDX Relay turns that proven behavior into a portable profile-driven tool without hiding the consequential parts. Users see the exact generated bytes and target paths before anything touches a repository.

## Who is it for?

Technical writers and developers who author in Obsidian, publish through Git-backed MDX sites, and want a repeatable workflow that is safer than a one-off conversion script.

## What does "done" look like?

V1 ships as a desktop-only Obsidian plugin with:

- One production profile matching the captured `dpw-mind-net` importer contract.
- Named repository/output profiles with validated paths and no executable configuration.
- Deterministic Markdown-to-MDX and PNG/JPEG/WebP conversion.
- A sealed preview showing every created or updated file and the exact MDX diff.
- Explicit approval before repository writes.
- Narrow staging, exact staged-byte verification, crash recovery, commit, push, and remote-tip verification.
- A macOS alpha followed by Windows 11 and Ubuntu validation before Community Plugin submission.

## Inspirations & References

- The existing `dpw-mind-net` Obsidian-to-MDX importer and its tests.
- Obsidian's desktop plugin model.
- Git's plumbing commands and compare-and-swap ref updates.
- Publishing tools that expose a reviewable plan instead of hiding deployment behind one button.

## Non-Negotiables

- Preview performs no plugin-requested repository mutation.
- The executor writes only sealed bytes that the user approved.
- Any changed source, dependency, target, branch, index, remote, profile, or Git configuration invalidates approval.
- Never use broad staging, force push, implicit pull/rebase, automatic conflict resolution, or deletion.
- No success state until the exact commit is observed at the effective push destination.
- Ambiguous paths, image resolution, Git state, or recovery state fail closed.
- Credentials never enter profiles, logs, plans, or repository files.
- Mobile is out of scope.
